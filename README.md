# AE2 Pattern Generator

AE2 Pattern Generator 是一个专为 Minecraft 1.7.10 (如 GregTech New Horizons 等注重复杂合成的模组环境) 开发的实用辅助模组。
其核心功能在于基于可配置的过滤系统与平衡机制，将 GregTech 机器合成配方批量导出为标准的 AE2 编码样板。

---

## ✨ 核心特性

### 1. 一次性批量生成样板
可按机器 `RecipeMap`（如 `assembler`、`implosion_compressor`）批量扫描配方，并转换为 AE2 `Processing Pattern`。物品输入/输出与流体输入/输出都在同一流程内处理。

### 2. 过滤规则够细，减少无用样板
生成前可通过多层过滤减少冗余：
- **矿辞正则匹配**: 对输入/输出的矿物词典条目做正则筛选。
- **黑名单过滤**: 按规则排除指定输入或输出。
- **强制替换**: 按配置将指定材料统一映射（例如 `ingotCopper` -> `dustCopper`）。
- **严格电压匹配**: 只保留与目标电压层级（如 `HV`、`EV`）一致的配方。

### 3. 支持从 NEI 直接拖拽填充筛选条件
所有筛选输入框都支持从 NEI 直接拖拽物品进行填充。拖入后会先按默认优先级自动写成显式语法；如果该物品同时存在多种可用信息，输入框右侧会出现一个轻量切换按钮，可在 `ID` / `矿辞` / `显示名` 之间切换。

### 4. 生成前先做消耗校验
生成请求会先校验 `Blank Pattern` 库存，再执行扣除。支持优先从已绑定的 ME 网络取材，不足时回退到玩家库存。数量不足会直接拒绝请求并给出提示。

### 5. 先构建配方缓存，再进行预览或生成
`Preview Count` 与 `Generate` 现在都只读取服务端持久化配方缓存。第一次使用、重启后，或模组/配置发生变化后，需要先在主界面点击 `Cache` / `构建缓存` 按钮。缓存构建会在后台执行，并通过聊天栏返回进度和统计信息。

### 6. 先写入虚拟仓储，再统一导出
生成结果先写入服务端虚拟仓储，不直接落地。可在 GUI 中查看、删除、清空，再批量导出到目标容器（箱子、AE2 接口等）。

### 7. 支持无线绑定和远程取材
支持与 AE2 网络无线绑定。完成绑定后，可在生成时直接使用网络内空白样板，减少手动物料搬运。

## 🎮 操作指南

| 交互方式 | 功能说明 |
|------|------|
| **右键**（对空气） | 打开**主配置终端** — 填写设备 ID、正则过滤规则、指定电压等并提交生成请求。 |
| **Shift + 右键**（对空气） | 打开**存储管理器** — 翻阅并管理当前存储在虚拟节点的样板数据。 |
| **Shift + 右键**（对方块） | **方块检测 / 条件导出** — 优先检测目标方块（含 GT 主方块配方信息）；若目标为可写入容器，则将虚拟样板批量导出到该容器。 |

此外，玩家可将光标悬停在物品上按下 `Shift` 键调出操作提示。系统同样提供用于控制台执行或 OP 自动化触发的命令节点：`/patterngen`。

首次使用、重启后，或模组 / 配置发生变化后，请先在主界面点击 `Cache` / `构建缓存`，待缓存完成后再使用 `Preview Count` 或 `Generate`。

## 🔎 过滤规则说明（统一显式语法）

在生成样板前，你可以通过多层过滤快速缩小目标配方。  
本模组的匹配机制统一遵循以下规则：

- **默认不区分大小写**
- **默认包含匹配**（类似 `find`，不是整串全等）
- **留空或填 `*` 视为不启用该条件**
- **多个过滤条件是 AND 关系**：必须全部通过才会保留配方

### 1. 所有筛选字段都使用同一套语法

以下这些字段现在完全共用同一套匹配逻辑：

1. 输出筛选
2. 输入筛选
3. NC 筛选
4. 输入排除
5. 输出排除

**必须使用括号明确标注类型**：

| 语法 | 含义 | 示例 |
|------|------|------|
| `[id]` | 按物品 ID 匹配，忽略 meta | `[8119]` |
| `[id:meta]` | 按物品 ID + meta 精确匹配 | `[8119:12]` |
| `(regex)` | 仅按矿辞匹配 | `(dustCopper)` |
| `{regex}` | 仅按显示名匹配 | `{Copper Dust}` |
| `*` | 关闭该字段的筛选 | `*` |

支持在同一字段中写多个显式 token，之间可用空格、`,`、`;` 或 `|` 分隔：

- `[8119] [8119:12]`
- `(dustCopper)|(dustTin)`
- `{铜粉} {Copper Dust}`

> 正则只对对应类型生效：`( ... )` 只查矿辞，`{ ... }` 只查显示名。

### 2. 匹配顺序与性能

统一显式语法的目的之一是避免在大配方表中过度解析物品信息：

1. `[id]` / `[id:meta]` 只比较 ID 和 meta，不查矿辞，不取显示名。
2. `(regex)` 只查询矿辞。
3. `{regex}` 才会解析显示名。

这样可以显著降低在 GTNH 大型配方表中使用筛选器时的卡顿风险。

### 3. 裸文本不再生效

- `dustCopper`
- `铜粉`
- `8119:12`

以上写法在这 5 个筛选字段里都**不会参与匹配**。如果要生效，必须改写成：

- `(dustCopper)`
- `{铜粉}`
- `[8119:12]`

### 4. 中文匹配说明（显示名）

- **机制上支持中文匹配**：`{...}` 内部使用 Java 正则，对中文字符可正常匹配。  
- **实际命中取决于服务端语言环境**：过滤在服务端执行，使用服务端拿到的显示名文本。  
- 若服务端显示名不是中文，你输入 `{铜粉}` 可能匹配不到。  
- 需要跨环境稳定匹配时，优先使用 **矿辞** 或 **`[id] / [id:meta]`**。  

### 5. 支持从 NEI 拖拽自动填充

以下 5 个筛选字段都支持把 NEI 里的物品直接拖到输入框内：

1. 输出筛选
2. 输入筛选
3. NC 筛选
4. 输入排除
5. 输出排除

拖入后会按以下优先级自动写入当前输入框：

1. 有 `id` 且有 `meta` 时，填入 `[id:meta]`
2. 只有 `id` 时，填入 `[id]`
3. 没有可用 `id` 但有矿辞时，填入 `(矿辞)`
4. 上述都没有时，填入 `{显示名}`

如果该物品还同时具备其他可用来源，右侧切换按钮会保留这些候选项：

- 左键切换到下一个来源
- 右键切换到上一个来源

这样既保留了拖拽即填充的速度，也允许你在默认值不合适时快速改成矿辞或显示名。
本方法适用于单个物品进行过滤，当需要使用正则时，更推荐手动填写。

## 🚀 部署与安装

1. 从 [Releases](https://github.com/Ch4oooooooLL/AE2PatternGen/releases) 页面获取最新的构建产物 (`.jar` 文件)。
2. 将其放置于客户端或服务端的 `mods` 路径下后启动应用。

---

## 🛠️ 构建源码

本项目基于 RetroFuturaGradle (RFG) 系统开发：

```bash
# 增量构建类文件
./gradlew classes

# 执行代码风格标准化 (修复 CI 潜在报错)
./gradlew spotlessApply

# 全量构建发布版本
./gradlew build
```
编译产物将被输出至 `build/libs` 目录下。
